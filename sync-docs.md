You are maintaining a centralized knowledge base of all Markdown documentation across the user's projects. Follow these steps precisely.

**Target directory**: If $ARGUMENTS is provided, use it as the scan root. Otherwise default to `/Users/cm/Downloads`.

**Output directory**: `/Users/cm/Downloads/claude-knowledge/` (create if it does not exist).

---

## Step 0: Load Previous State

Read `/Users/cm/Downloads/claude-knowledge/hashes.json` if it exists. This contains the previous scan's file hashes for change detection. If it does not exist, this is a fresh scan — treat all files as new.

## Step 1: Discover Files

Use the Bash tool to find all candidate MD files:

```bash
find <TARGET_DIR> -name "*.md" -type f \
  -not -path "*/node_modules/*" \
  -not -path "*/.venv/*" \
  -not -path "*/.git/*" \
  -not -path "*/dist/*" \
  -not -path "*/build/*" \
  -not -path "*/.cache/*" \
  -not -path "*/__pycache__/*" \
  -not -path "*/.expo/*" \
  -not -path "*/.pytest_cache/*" \
  -not -path "*/.github/agents/*" \
  -not -path "*/.github/prompts/*" \
  -not -path "*/.specify/templates/*" \
  -not -name "LICENSE.md" \
  -not -name "LICENCE.md" \
  2>/dev/null | sort
```

Save the full file list.

## Step 2: Hash All Files

Compute MD5 hashes for all discovered files. Use macOS native `md5 -r`:

```bash
md5 -r <file1> <file2> ... <fileN>
```

If the file list exceeds ~100 files, batch into groups of 100 per command to avoid argument-length limits. Build a mapping of `path → hash`.

## Step 3: Detect Changes

Compare new hashes against `hashes.json` from Step 0:

- **New**: path exists now but not in previous hashes
- **Updated**: path exists in both but hash changed
- **Deleted**: path in previous hashes but no longer exists
- **Unchanged**: same path, same hash

Track these four lists. Only new and updated files need content reading in Step 6.

## Step 4: Deduplicate

Group files by their MD5 hash. For any hash appearing more than once:

1. Pick a **canonical** path: prefer files in a project that has a CLAUDE.md (active project), then prefer shorter paths, then prefer `docs/` subdirectories.
2. Other paths with the same hash become **aliases**.

Known duplicate sets to expect:
- `couple_rate/docs/experience/*` = `Pixel_video/experience/*`
- `Auto_Vid_Gen/**` = `CM-social-platforms/**`

For files with the same filename but different content (e.g., multiple `CLAUDE.md`), these are NOT duplicates. Just note them in the report for user awareness.

## Step 5: Categorize

For each unique (non-alias) file, assign a category. Check rules in order, first match wins:

| Category | Match Rules |
|---|---|
| Engineering Lessons | Path contains `experience/`, or filename matches `*bugs*`, `*lessons*`, `*pitfalls*` |
| Architecture | Path contains `architecture/`, or filename matches `*ARCHITECTURE*`, `*-design*` |
| Product Specs | Path contains `specs/`, or filename matches `*PRD*`, `*spec*`, `*usecase*`, `*journey*`, `*Product*` |
| Security | Filename matches `*security*`, `*audit*`, `*SECURITY*` |
| Dev Guides | Filename is `CLAUDE.md`, `DEV_GUIDE*`, `INITIALIZE*`, `development.md`, `SERVER.md`, or path contains `docs/requirements/` |
| Planning | Path contains `planning/`, or filename matches `*ROADMAP*`, `*PROGRESS*`, `*plan*` |
| Project Config | Filename is `SKILL.md`, `COMMAND.md` |
| Other | Everything else |

## Step 6: Build context.md

For each **new or updated** file, use the Read tool to read the first 800 characters. Extract:
- The title (first `# heading` line)
- A one-line takeaway: the core lesson, decision, or fact

For **unchanged** files, reuse their entry from the existing `context.md` (read and parse by path).

Write `/Users/cm/Downloads/claude-knowledge/context.md` with this structure:

```markdown
# Cross-Project Knowledge Base
> Auto-generated by /sync-docs on YYYY-MM-DD HH:MM
> Scan root: <TARGET_DIR>
> Files indexed: N unique (M total, D duplicates)

---

## Engineering Lessons

### [Title] — project-name
> Source: relative/path/to/file.md

One-line takeaway.

---

## Architecture

### [Title] — project-name
> Source: relative/path/to/file.md

One-line takeaway.

---

(continue for all categories)
```

**Size constraint**: context.md must stay under 2000 lines. If it exceeds:
1. Drop "Other" category entries first
2. Then drop "Project Config" entries
3. Then truncate remaining to just title + takeaway (no metadata)
